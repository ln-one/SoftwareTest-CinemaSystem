# 电影院管理系统测试实验分工方案

## 项目信息

- 项目名称：电影院售票管理系统 (Cinema Manager)
- 技术栈：Spring Boot 2.4 + MyBatis + MySQL
- 测试目标：Service 层单元测试，统计语句覆盖率和条件覆盖率

## 团队分工（5人）

每位成员负责自己模块的：测试用例编写 + 覆盖率统计 + 模块报告撰写

| 成员 | 测试模块 | 测试类 |
|------|---------|--------|
| 成员 A | 用户模块 | SysUserServiceTest |
| 成员 B | 订单模块 + 场次模块 | SysBillServiceTest, SysSessionServiceTest |
| 成员 C | 电影模块 + 类别模块 | SysMovieServiceTest, SysMovieCategoryServiceTest |
| 成员 D | 影院模块 + 影厅模块 | SysCinemaServiceTest, SysHallServiceTest |
| 成员 E | 角色权限模块 | SysRoleServiceTest, SysResourceServiceTest |

## 各成员任务详情

每位成员需完成以下三项工作：

### 成员 A：用户模块

**1. 测试用例编写**
- 用户注册（密码加密、用户名唯一性校验）
- 用户登录（密码验证、Token生成）
- 用户增删改查

**2. 覆盖率统计**
- 运行 SysUserServiceTest，截图语句覆盖率和条件覆盖率

**3. 模块报告**
- 用户模块测试用例说明
- 测试执行结果截图
- 覆盖率分析
- 失败用例 Bug 根因分析（如有）

## 公共任务

| 任务 | 负责人 | 说明 |
|------|--------|------|
| JaCoCo 环境配置 | 全员协作 | 配置 pom.xml，确保覆盖率工具可用 |
| 报告整合 | 轮流负责 | 将 5 份模块报告合并为完整实验报告 |
| 项目介绍章节 | 全员协商 | 简要介绍项目背景（可复用 README） |
| 总结章节 | 全员协商 | 汇总各模块测试结论 |

## 时间规划

| 阶段 | 时间 | 任务 |
|------|------|------|
| 环境搭建 | 12/17 - 12/18 | 全员配置 JaCoCo，熟悉各自模块代码 |
| 测试编写 | 12/19 - 12/24 | 各自编写负责模块的测试用例 |
| 覆盖率统计 | 12/25 - 12/26 | 各自运行测试，截图覆盖率 |
| 报告撰写 | 12/27 - 12/29 | 各自完成模块报告 |
| 整合Review | 12/30 | 合并报告，全员检查 |
| 提交 | 12/31 | 最终提交 |

## 最终报告结构

